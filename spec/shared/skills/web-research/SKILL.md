# Web research

Apply this method whenever an answer depends on facts you cannot verify from
your own knowledge, or that may have changed recently.

## Method

1. **Decompose the question.** Identify the specific, checkable facts the answer
   depends on. Separate what you already know from what must be verified.
2. **Search deliberately.** Issue focused `web_search` queries of 3-10 words,
   one fact at a time. Prefer specific terms (names, dates, identifiers) over
   broad phrases. Run a second query with different wording if the first is
   thin.
3. **Corroborate.** Treat a single source as provisional. For anything
   contested, time-sensitive, or numerical, confirm it against a second
   independent source before stating it.
4. **Attribute.** When you state a verified fact, name the source so the user
   can check it. Quote exact figures and dates rather than paraphrasing them.
5. **Mark gaps.** If searches do not settle a point, say so plainly, give your
   best current understanding, and state exactly what would confirm it.

## Guardrails

- Do not present an unverified claim as verified. Distinguish "sources say" from
  "I believe."
- Prefer primary and recent sources over aggregators when they conflict.
- Keep the final answer concise: lead with the verified conclusion, then add
  the supporting sources and any caveats.
